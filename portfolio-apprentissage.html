🖥️ Serveur Web & Dossier Partagé – Machine Virtuelle
BUT 1 · IUT Dijon · 2023-2024
Majeure C3 (RACDV)

Projet d'administration système réalisé sous VirtualBox avec une machine virtuelle Ubuntu Server. L'objectif était de déployer un environnement web complet (XAMPP), d'y héberger une application PHP accessible depuis la machine hôte, et de configurer un dossier partagé multi-utilisateurs avec gestion fine des permissions. Ce TP m'a confronté pour la première fois à l'administration réelle d'un serveur Linux de bout en bout.

// Preuves & traces d'apprentissage

  • Création et configuration de la VM : installation d'Ubuntu Server 22.04 LTS sur VirtualBox, configuration de la carte réseau en mode Bridge pour rendre le serveur accessible depuis le réseau local (adresse IP fixe attribuée via /etc/netplan).
  • Déploiement de XAMPP : installation de la stack XAMPP (Apache 2.4, MariaDB, PHP 8.1) via le script .run, démarrage des services (sudo /opt/lampp/lampp start), vérification de l'accessibilité du serveur Apache depuis le navigateur hôte via l'IP de la VM.
  • Hébergement d'une application web : dépôt d'un projet PHP dans le répertoire /opt/lampp/htdocs/, test de rendu depuis le navigateur de la machine hôte à l'adresse http://192.168.x.x/monprojet.
  • Création d'un dossier partagé : mise en place d'un répertoire /srv/partage/ accessible par plusieurs utilisateurs Linux (alice, bob, invite), avec configuration d'un groupe dédié (groupadd equipe, usermod -aG equipe alice).
  • Gestion fine des autorisations : application des permissions chmod 2770 (SGID pour que les nouveaux fichiers héritent du groupe), distinction des droits lecture/écriture/exécution par utilisateur et par groupe, vérification avec ls -la et tests croisés entre sessions utilisateurs.
  • Sécurisation des accès : désactivation du compte root SSH, création de comptes utilisateurs avec mots de passe hachés, restriction des accès au dossier partagé pour l'utilisateur invite (lecture seule via chmod 750).

// Compétences techniques & humaines mobilisées

VirtualBox Ubuntu Server XAMPP (Apache/PHP/MariaDB) Linux CLI (Bash) chmod / chown / SGID Gestion utilisateurs Linux Configuration réseau (netplan) SSH Rigueur & Méthodologie

⚙️ Moteur de Matchmaking – Optimisation C#
BUT 1 · IUT Dijon · 2023-2024
Majeure C4 (RACDV)

Analyse de performance et réécriture d'un algorithme de matchmaking en C#. Le projet consistait à analyser des données de joueurs (classements, statistiques), identifier les goulots d'étranglement algorithmiques, et optimiser les structures de données pour réduire la complexité temporelle.

// Preuves & traces d'apprentissage

  • Analyse de données joueurs : traitement et tri de larges ensembles de données (scores ELO, historiques de parties) pour alimenter l'algorithme de matching.
  • Optimisation des structures : remplacement de listes chaînées par des dictionnaires et tables de hachage (Dictionary<T>) pour passer de O(n²) à O(n log n) sur les opérations critiques.
  • Mesure de performance : utilisation de Stopwatch et profiling pour comparer les temps d'exécution avant/après optimisation, avec production de graphiques de complexité.
  • Documentation technique : rédaction d'un rapport d'analyse avec tableaux de comparaison de performance, choix de structures justifiés et recommandations.

// Compétences techniques & humaines mobilisées

C# / .NET Complexité algorithmique Data Analyse Profiling & Benchmarking Structures de données Prise de décision Documentation

🌙 Nuit de l'Info – Application React.js
BUT 1 · IUT Dijon · Décembre 2023
Majeure C5 (RACDV)

Événement compétitif national : développement d'une application web complète en une seule nuit (8h) en équipe. Contrainte temps extrême, déploiement en production obligatoire. Ce projet a exigé une gestion de projet agile intensive : priorisation constante, découpage en tâches, coordination en temps réel.

// Preuves & traces d'apprentissage

  • Gestion du temps sous contrainte : découpage du projet en jalons horaires (design 30 min, structure 1h, features 4h, déploiement 30 min) pour garantir une livraison opérationnelle.
  • Coordination d'équipe en temps réel : attribution claire des rôles (UI, logique métier, déploiement), communication continue via Discord, revue de code informelle.
  • Priorisation des fonctionnalités : identification du MVP (Minimum Viable Product) dès le départ pour garantir un résultat livrable même en cas de dépassement de temps.
  • Déploiement en production : mise en ligne de l'application React/Vite sur un hébergeur en fin de nuit – livraison effective et fonctionnelle dans les délais.

// Compétences techniques & humaines mobilisées

React.js / Vite JSX Déploiement web Gestion sous contrainte Travail en équipe Priorisation (MVP) Créativité Résilience

🌐 Plateforme Web PHP – Scrum & Besoin Client
BUT 2 · IUT Montpellier · 2024-2025
Majeure C5 (RACDV)

Projet mené en mode agile Scrum sur plusieurs mois avec un client fictif. J'ai occupé le rôle de Scrum Master : animation des cérémonies Scrum, rédaction des user stories, planification des sprints et gestion du backlog produit.

// Preuves & traces d'apprentissage

  • Recueil et formalisation des besoins : conduite d'entretiens avec le client, rédaction de user stories structurées (En tant que… Je veux… Afin de…) avec critères d'acceptation.

// Compétences techniques & humaines mobilisées

Scrum / Agile Trello / Jira User Stories Planning Poker Leadership Communication client Gestion des risques Organisation
