Projet 2 - BUT 1

Optimisation Algorithmique

Analyse de performance et implémentation de structures de données sur du code Legacy.

Le Défi Technique

Le but de cette SAÉ était d'intervenir sur une application existante en C# pour y ajouter une fonctionnalité de gestion de recettes de cuisine, tout en optimisant les performances globales.

La contrainte majeure : Interdiction de modifier le code existant (Legacy Code) en dehors des points d'extension prévus. Il fallait également n'utiliser aucun Framework externe pour comprendre les mécanismes bas niveau.

Approche et Méthodologie

1. Analyse Statistique

Avant d'écrire la moindre ligne de code, j'ai mené une phase de recherche de données statistiques pour comprendre les cas d'utilisation fréquents de l'application. "Optimiser à l'aveugle est une erreur".

2. Choix des Structures de Données

J'ai dû arbitrer entre différentes structures de données. Mon choix s'est porté sur les Listes Doublement Chaînées (Doubly Linked Lists) pour la gestion des recettes.

  • Réalisation de diagrammes comparatifs pour visualiser les gains de performance.
  • Benchmark entre Tableaux statiques et Listes chaînées.

Ce choix permettait des insertions et suppressions en O(1) une fois l'élément localisé, ce qui était crucial pour notre cas d'usage, contrairement aux tableaux nécessitant des décalages coûteux en mémoire.

Résultats et Apprentissages

Le projet a abouti à une application plus performante, accompagnée d'un dossier technique justifiant chaque décision par des preuves chiffrées.

J'ai appris à :

  • Coder proprement en respectant des conventions strictes.
  • Utiliser GitHub de manière collaborative (gestion des conflits, branches).
  • Planifier une phase de conception avant l'implémentation.
  • Comprendre qu'une optimisation ne vaut que si elle est mesurable.
