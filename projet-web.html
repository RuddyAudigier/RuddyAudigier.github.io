Projet site web - BUT 2

Développement Web & Méthode Agile

Conception d'une application client complète avec focus sur la sécurité et la maintenabilité.

Contexte et Enjeux

Ce projet s'inscrit dans une commande académique simulant une situation professionnelle réelle. En équipe de 4, nous devions livrer au client une application web fonctionnelle connectée à une base de données relationnelle.

Les contraintes techniques étaient strictes : utilisation native de PHP, implémentation d'un framework CSS, et application rigoureuse de la méthodologie Scrum.

Ma Contribution et mes Responsabilités

1. Gestion de Projet & Relation Client

J'ai endossé le rôle clé de Product Owner. Ma mission consistait à faire l'interface entre mon équipe et le client :

  • Organisation et conduite des réunions avec le client.
  • Collecte des besoins et clarification des questions de mon équipe sur le projet.
  • Traduction des besoins clients en spécifications techniques claires (User Stories) pour faciliter l'implémentation.

J'ai également agi en tant que Scrum Master sur un sprint complet. Ma mission consistait à faire respecter la méthode Scrum à mon équipe :

  • Distribution des tâches selon les forces de chacun.
  • Organisation de réunion quotidienne (Daily meeting) afin de suivre l'avancement de chacun.
  • Relever les problèmes rencontrées par les membres de l'équipe sur leurs tâches respectives.

2. Architecture Technique & Qualité du Code

Sur le plan technique, je me suis concentré sur la robustesse du backend :

  • Refactoring & Clean Code : J'ai restructuré le code après chaque sprint pour respecter les principes SOLID et rendre l'application évolutive. Utilisation de Design Patterns pour découpler les composants.
  • Sécurité Avancée : Implémentation d'un système d'authentification robuste avec hachage des mots de passe (technique du "Sel et Poivre") et gestion des droits d'accès (RBAC) selon les rôles utilisateurs.
  • Optimisation BDD : Restructuration du schéma SQL et ajout d'index stratégiques pour accélérer les temps de réponse des requêtes complexes.

Compétences Acquises

Ce projet m'a appris à :

  • Gérer la pression et les deadlines strictes inhérentes aux projets longs.
  • Discuter et négocier avec le client en fonction de ses demandes.
  • Retranscrire les besoins du client à mon équipe de manière compréhensible et claire.
  • Comprendre l'intérêt vital des principes SOLID : un code propre aujourd'hui est un code maintenable demain.
  • Maîtriser les bonnes pratiques de versionning sous GitLab (Merge Requests, Code Review, Refactoring de branche).
  • Sécuriser une application web contre les failles courantes.
